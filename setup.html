Set up your Stockkar Algo app in a few simple steps.

Follow the video, pick any VPS (or AWS / Oracle), connect your broker, and start using Stockkar.

Easiest · Any host

One-command install (any VPS)

Works on any Ubuntu server — a cheap $3 VPS (InterServer), Azure free tier, DigitalOcean, Hetzner, anything. Create a small Ubuntu VPS, then paste this one line over SSH:

curl -fsSL https://raw.githubusercontent.com/mindvisualmedia-jpg/Stockkaralgo/main/scripts/install.sh | sudo bash

Installs everything (Node, app, HTTPS, one-click auto-updater) and prints your app URL. ~2 minutes.

One click · Azure

Deploy to Azure (free tier)

Creates the whole server for you on Azure — network, firewall, public IP, and an Ubuntu VM — then runs the installer automatically. No region, disk, or zone to pick. You only choose an app name and paste your SSH key.

After it finishes (~3–5 min), open the deployment Outputs tab for your app URL. Free for 12 months on B2ats_v2 / B2pts_v2.

AWS Free Tier

Download the AWS setup file, open AWS CloudFormation, upload the file, then enter your app name, private update PIN, and alert email inside AWS.

AWS does not accept normal website links as CloudFormation template URLs, so this upload method is the reliable setup path.

Steps To Follow

1

Watch the setup video

Watch the complete walkthrough once before starting. AWS and Oracle each have their own setup path.

2

Create or open your cloud account

Choose AWS or Oracle. Your app, broker connection, and static IP stay under your own account.

3

Upload the setup file

For AWS, upload the YAML file in CloudFormation. For Oracle, use the ZIP file with Resource Manager.

4

Enter app name, PIN, and email

Use a simple app name, set a private update PIN, and enter your email for alerts. Leave other settings unchanged.

5

Open your Stockkar app link

When setup completes, open Outputs and click AppUrl. That is your personal Stockkar link on the selected cloud.

6

Connect broker and start

Paste the static IP in your broker account, save tokens inside Stockkar, fetch screener, configure algo, and start monitoring.
